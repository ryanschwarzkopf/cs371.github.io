Binary Search Tree

Component Testing

For each function in the binary tree, we did a test to see if it works correctly. There are four functions in the binary tree, which are Create, Insert, Search, Remove Every function is functioning through our test cases.

-Create -Testing create function in two cases First, see if Create will function when there's no tree, Second, see if Create will function when there's a tree already. For the first case, the expected function result is creating a new tree. For the second case, the expected function result is to remove everything in the old tree, and create a new one.

-Insert -Testing create function in two cases First, see if Insert will function when there's no tree, Second, see if Insert will function when there's a tree already. For the first case, the expected function result is to insert no value because the tree has to be created first. For the second case, the expected function result is to insert a new node, and display the change.

-Search -Testing create function in two cases First, see if Search will function when there's no tree, Second, see if Search will function when there's no node found, Third, see if Search will function when nodes are found. For the first case, the expected function result is sending a message of "there's no tree to find a node". For the second case, the expected function result is sending a message of "the node is not found". For the third case, the expected function result is sending a message of "the node is found", and displaying the index.

-Remove -Testing create function in two cases First, see if Remove will function when there's no tree, Second, see if Remove will function when there's no node found, Third, see if Remove will function when nodes are found. For the first case, the expected function result is sending a message of "there's no tree to remove a node". For the second case, the expected function result is sending a message of "the node is not found". For the third case, the expected function result is sending a message of "the node is found, and removed", and remove the node from the tree.

Component Testing

No unit testing tools were used when testing the Stack data structure. The main test that was used was making sure the user input was displaying prop- erly. The way we tested this was inputing number and making adjustments to the code as needed. I also tested that each function properly updated each value on the stack when each function was being used.

System Testing

The data used for system testing were random single digit and double digit numbers. The first test was the Push function. This function had an issue that it was not updating the website in real time. The solution to this was creating a delay and making the website refresh every one second. Each function went through the same test cases with random numbers. When testing the Top function, it was difficult to decide how to demonstrate it. The solution was to highlight the top most element on the stack which resulted in one issue. This error happened because the print statement was incorrectly displaying the value that was on the top of the stack. This was a simple fix that did not cause any other issues between the other functions.
